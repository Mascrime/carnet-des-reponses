Power BI Dataflow Migration & Duplication Strategies

Publié le 2025-08-19

Workflow: Create Your "Master" Dataflow

First, build the complete dataflow in the Power BI Service with all your queries, functions, and parameters (using Method 1: Manual Copy & Paste).

Export the JSON Model

In your workspace, find the completed dataflow. Click the ellipsis (...) next to the dataflow name and select "Export .json". This will download a model.json file that contains the M code for all queries and the entire structure of your dataflow.

Import to Create a New Dataflow

Go to the workspace where you want the new dataflow (it can be the same or a different one). Click "New" -> "Dataflow". At the top of the "Add new tables" screen, you will see an "Import model" link. Click it. Select the model.json file you just downloaded. Done! Power BI will instantly create a new dataflow with all the queries, folders, and parameters from your exported file. You will just need to configure the credentials and schedule the refresh.

Summary & Recommendation

Here's a breakdown of recommended methods for different dataflow management tasks:

Task Recommended Method Why?
Initial migration from a .pbix file to your first Dataflow Method 1: Manual Copy & Paste It's the only direct way. It's tedious but ensures a 1-to-1 transfer of your logic.
Duplicating an existing Dataflow or creating a template Method 2: Import/Export Model This is the true "all at once" solution for replicating dataflows, saving immense amounts of time.
Moving a Dataflow from a Dev to a Prod workspace Method 2: Import/Export Model This is the standard best practice for CI/CD (Continuous Integration/Deployment) with dataflows.

So, for your initial task, you'll need to use the manual copy-and-paste method. It's a one-time effort, and once it's done, you can use the much faster Import/Export method for any future needs.
